# 🔒 Privacy & Memory

### *Your Data. Your Control.*

Understanding how TechPixel Agent (v1.2) handles conversations, memories, training submissions, and personalization.

# 🛡 Privacy Philosophy

TechPixel Agent aims to provide personalized assistance while giving users meaningful control over their information.

The platform is designed around the following principles:

- 🔍 **Transparency** — Explain what features use your information.
- ⚙️ **Control** — Allow users to manage or remove their data.
- 🧠 **Personalization** — Improve experiences only when beneficial.
- 🚪 **Accessibility** — Support both guest and signed-in experiences.
- 🤝 **Trust** — Clearly communicate limitations and capabilities.

---

# 🧠 Understanding Memory

One of TechPixel's optional features is **Dynamic Memory**.

Dynamic Memory allows TechPixel to remember information across conversations to create a more personalized experience.

For example:

> "Remember that I prefer concise explanations."

> "Remember that I'm building a prison server."

> "Remember that I mostly work with WorldCode."

This information helps reduce repetition and allows TechPixel to better understand your preferences over time.

---

# ✨ What Memory Can Remember

Examples of information that may be stored include:

## Communication Preferences

- Preferred explanation style
- Desired response length
- Formatting preferences
- Teaching preferences

Examples:

```text
Use simpler explanations.

Give detailed examples.

Use bullet points whenever possible.
```

---

## Project Context

Information related to ongoing Bloxd projects.

Examples:

```text
I'm creating a prison server.

My project focuses on PvP.

I use WorldCode extensively.
```

---

## Workflow Preferences

Patterns that improve future interactions.

Examples:

```text
I prefer Research Mode for large questions.

I usually export conversations as PDFs.

I often ask for code optimization.
```

---

# 🚫 What Memory Is Not Intended For

Dynamic Memory exists to improve convenience.

It is **not intended to function as permanent storage for everything you say.**

You should avoid relying on memory for:

- Long-term backups
- Critical record keeping
- Sensitive personal information
- Information that requires guaranteed persistence

If information is important, consider exporting conversations separately.

---

# 🗑 Managing Memory

You remain in control of your stored memories.

You can instruct TechPixel to modify memory using natural language.

Examples:

## View Preferences

```text
What do you remember about me?
```

---

## Delete Specific Memories

```text
Forget that I prefer concise explanations.

Delete that memory.
```

---

## Clear All Memories

```text
Forget everything you know about me.
```

---

Memory controls are designed to be straightforward and accessible.

# 👤 Guest Sessions

TechPixel supports Guest Sessions.

Guest Sessions are intended for users who want immediate access without signing in.

Guest Sessions provide:

- Instant access
- Feature exploration
- Quick experimentation
- Low-friction onboarding

---

## Guest Session Considerations

Guest Sessions may not provide the same persistence available to signed-in users.

Examples include:

- Conversations may not be retained.
- Personalization may be limited.
- Preferences may not persist between sessions.

Guest Sessions prioritize accessibility over continuity.

TechPixel may retain conversations to provide features such as:

# 📄 PDF Exports

TechPixel allows conversations to be exported into PDF documents.

PDF exports are useful for:

- Documentation
- Knowledge sharing
- Personal archives

## Export Responsibility

Once exported:

> The resulting PDF becomes your responsibility to manage and protect.

TechPixel cannot control how exported files are shared, distributed, or stored externally.
